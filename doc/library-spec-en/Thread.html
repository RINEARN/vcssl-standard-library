Thread

Abstract

The VCSSL Thread library is a standard library for multithreading.

- Note -
The current version of RINEARN VCSSL Runtime supports the VCSSL Thread API, but its support for multithreaded processing is not yet sufficient. This is because the basic design of the current VCSSL engine still partially follows the design of the early VCSSL engine, which had been developed assuming only single-threaded execution. As a result, depending on the processing contents, some inconveniences involving logical inconsistencies may occur. If you are considering using the VCSSL Thread API in RINEARN VCSSL Runtime, please make that decision with this situation in mind. The following are examples of issues that may currently occur.
- Index inconsistencies when multiple threads access the same array variable at the same time
In the current VCSSL engine, modules that refer to the values of array variables still include designs that assume only single-threaded execution. Because of this, if the same array variable is accessed simultaneously, incorrect index values may be referenced.
- Performance bottlenecks
The current VCSSL engine includes multiple points that become bottlenecks in multithreaded processing. Therefore, even if highly parallel work is divided among multiple threads, a major improvement in performance cannot be expected. In other words, the Thread API in RINEARN VCSSL Runtime is useful for creating multiple flows of processing, but it is not very meaningful for the purpose of extracting computational performance.
- Emergency engine stop caused by very frequent simultaneous access from multiple threads to the same function
In principle, the same function can be executed by multiple threads. In that case, arguments and local variables are allocated separately for each thread so that they are not mixed up across threads. However, if simultaneous accesses are concentrated within extremely short intervals, this handling may fail to keep up, and the engine may stop itself in an emergency to prevent inconsistencies. Note that overlap in the execution time of the function itself is not the problem. The problem occurs when the moments of calling the function, that is, the moments of passing arguments, overlap.
Because inconveniences such as the above may occur, use of the VCSSL Thread API is currently not recommended in RINEARN VCSSL Runtime for processing that requires strict correctness, such as scientific and technical calculations or numerical computations.
